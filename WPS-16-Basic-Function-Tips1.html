那么怎么给我们自己的函数添加-whatif，-confirm之类的功能呢？将[CmdletBinding()]添加在param()前面并设置SupportsShouldProcess = $true:

Ensure your script or function declares the [CmdletBinding()] attribute, and then access engine features through the $psCmdlet automatic variable.

  • whatif: 告诉用户执行命令后会产生什么结果，但不会真的执行
  • confim：根据用户的反馈决定下一步的行动（执行，还是不执行..）
  • verbose: 输出write-verbose 命令打出的详细信息（看下面例子就知道了）

但是有时候用户他忘了或者不知道这些参数，但是对于危险的类似删除的action我们仍旧希望能够根据用户反馈来执行怎么办呢？这时候可以调用另一个方法(注意这个不要和上面的那个方法一起用)：

If your command causes a high-impact result that should be evaluated with caution,call the $psCmdlet.ShouldContinue() method. This generates a warning for users—but be sure to support a -Force parameter that lets them bypass this message.

Begin{} Process{} End{} with |

管道中的对象是顺序进行的，一般情况下多个管道时只有前一个管道内的命令执行完，才会执行下一个管道的命令：

但是当我们使用Begin{} Process{} End{}时则会改变这一运行机制：

  • 增加了代码可读性，能够更加清晰的明白函数的运行过程
  • 每个管道内只有一个函数的实例
  • 多个管道同时执行，或者应该说前一管道一旦产生对象到管道中，下一个会立即执行它，而不用等到上一管道内的命令完全结束

比较有趣的一点是Begin{} Process{} End{}作用域里面的变量是可以被访问和更改的。

Unlike most blocks of code contained within curly braces, the code in the begin, process, and end blocks has access to variables and functions defined within the blocks before it.

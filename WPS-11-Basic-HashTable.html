今天讲下哈希表(hashtable).
哈希表,可以理解为键值对的数组,里面的每个元素都是一对,由key和value组成.
一般哈希表的声明和赋值如下所示,@{}为主体,里面用等号=分隔开的键值对,如果写在一行要用;分隔开:

详细使用见微软官网About Hash Tables

其中一句Beginning in PowerShell 3.0, you can use the [ordered] attribute to create an ordered dictionary (System.Collections.Specialized.OrderedDictionary) in PowerShell.，这个偶尔会有需求创建有序哈希表。

To create a hash table, follow these guidelines:

  • Begin the hash table with an at sign (@).
  • Enclose the hash table in braces ({}).
  • Enter one or more key/value pairs for the content of the hash table.
  • Use an equal sign (=) to separate each key from its value.
  • Use a semicolon (;) or a line break to separate the key/value pairs.
  • Key that contains spaces must be enclosed in quotation marks. Values must be valid PowerShell expressions. Strings must appear in quotation marks, even if they do not include spaces.
  • To manage the hash table, save it in a variable.
  • When assigning an ordered hash table to a variable, place the [ordered] attribute before the “@” symbol. If you place it before the variable name, the command fails.

如果我们需要获取其中某个键(key)对应的值(value),直接像访问属性一样即可,也可以用方法获取

更多的属性和方法可以用get-member查看,比如查看是否包含某个key或者value,添加或者清空哈希表

这里我们主要讲下添加新的键值对和哈希表的遍历.

我们注意到刚才查看属性和方法的时候,对于Add,其参数key和value是object,是所有子集对象的父对象,也就是说key和value可以是任何类型的值.

Add Method void Add(System.Object key, System.Object value), void IDictionary.Add(System.Object key, System.Object value)

但是实际应用中我们一般还是给key赋值string的对象,而对于value,则可以多种多样,比如int,datetime,甚至hashtable.

正如之前一直强调过的powershell是完全基于.NET平台的,所以也可以用.NET方法创建和使用hashtable. 别的使用方法和上面说的都一样.

当不知道怎么使用一个对象的时候,一定要记得先用get-member查看其属性和方法,里面会有详细的使用说明. 比如一个方法需不需要参数,需要什么类型参数,返不返回参数,返回什么类型参数….

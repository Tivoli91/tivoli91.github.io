Windows PowerShell进阶(30) 未安装Excel情况下操作Excel文件
发表于 2019-09-03 | 更新于 2019-09-03
powershell
进阶 | IO | Excel | .NET

本文首发于本人公众号SPPS178,现搬运过来,较原文有所改动.

我们在第28篇结尾的时候提了下还可以在没有安装微软Excel的情况下操作Excel文件。这个是我在工作的时候有遇到这种情况，公司的windows server上是没有安装Excel的，但是我会有个PowerShell脚本在server上运行去抓取数据库的用户信息，每一类信息为单独的csv，然后要把这些csv合并到excel中去，每个csv保存在一个sheet里面。

然后面向Google编程了下,在Github上找到了一个PowerShell Module ImportExcel，可以实现我这需求。截止到今天这篇文章（ 2019/08/31）为止，我看到这个项目仍旧在维护和更新。

This PowerShell Module allows you to read and write Excel files without installing Microsoft Excel on your system. No need to bother with the cumbersome Excel COM-object. Creating Tables, Pivot Tables, Charts and much more has just become a lot easier.

个人分析

之前有提到过这其实就是用的.NET底层的方法，我们一步一步来看。

首先这个module安装在你的document文件下“C:\Users`SPPS`\Documents\WindowsPowerShell\Modules\ImportExcel\5.0.0”，把SPPS改成你自己用户名。

其主入口文件是ImportExcel.psm1，我们打开这个文件，可以看到在加载这个ImportExcel模块时，首要做的就是先加载EPPlus.dll这个DLL文件。


30-3 Check Module

那么至此我们已经可以肯定这个module能够在没有安装excel的情况下操作其对应的文件就是通过这个DLL来实现的。

我们再来看这个Export-Excel.ps1脚本，其接受一个[OfficeOpenXml.ExcelPackage]类型$ExcelPackage变量来最终保存Excel数据。接着到565行(行数可能会在这个项目后续更新至变化,大家知道意思即可)，我们可以看到，如果用户没有提供$ExcelPackage变量的情况下，脚本会创建一个OfficeOpenXml.ExcelPackage类型的变量$pkg = New-Object OfficeOpenXml.ExcelPackage $Path


30-4 Check Export Excel

那么我们可以想到EPPlus.dll这个DLL文件里面有一个类就是OfficeOpenXml.ExcelPackage，打开上节提到的反编译工具ILSpy打开EPPlus.dll可以看到和我们预料的完全一致。可以在右侧看到这个类的相关信息，像是最大行数/列数都在。

函数先讲到这，接着说New-Variable的一些用法，我们平时其实用这种$str=123变量赋值其实完全足够了，New-Variable个人觉得这个函数是给变量增加更多的约束，而这个用$的方式是很难实现的。

默认情况下，变量可被当前作用域和其子作用域访问，但只能被当前作用域更改。在脚本中，函数的作用域就是子作用域。在下面示例可以看到$number可以被Test-VariableScope函数访问，但是在函数里面更改的值并没有作用到其父作用域。

但是有些情况下我们为了安全起见甚至不希望子作用域可以访问父作用域的变量的值，这时候就可以用New-Variable来控制私有变量，可以看到在控制台中，函数里面的代码并没有打出来私有变量$count的值：

更甚至，有时候比如我们对于圆周率π，希望定义一个常量$pie=3.14,而且希望在接下来这个变量在任何作用域都不允许被改动，这个在java中有public final;在C#中有public const，那么在powershell中怎么定义呢？

除了Constant，Private还可以设置以下的options：

  • None. Sets no options. (None is the default.)
  • ReadOnly. Can be deleted. Cannot be not changed, except by using the Force parameter.
  • Private. The variable is available only in the current scope.
  • AllScope. The variable is copied to any new scopes that are created.
  • Constant. Cannot be deleted or changed. Constant is valid only when you are creating a variable. You cannot change the options of an existing variable to Constant.

其中关于作用域也可以通过以下参数设置

  • Global
  • Local
  • Script
  • A number relative to the current scope (0 through the number of scopes, where 0 is the current scope and 1 is its parent). Local is the default.

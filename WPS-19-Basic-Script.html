终于要开始讲script了，我们将之前所讲过的各种知识点保存在特定后缀名的文件里就是一个powershell script了，所以我的理解script本身也是一个scriptblock。

后缀名

powershell有以下几种后缀名，分别对应不同的应用场景，目前我们只要用到.ps1这个，以后的碰到了再细说。

文件扩展名(File extensions)

  • PS1 - Windows PowerShell外壳脚本
  • PSD1 - Windows PowerShell数据文件
  • PSM1 - Windows PowerShell模块文件
  • PS1XML - Windows PowerShell格式和类型定义
  • CLIXML - Windows PowerShell序列化数据
  • PSC1 - Windows PowerShell控制台文件
  • PSSC - Windows PowerShell会话配置文件

以下是运行脚本的一些方法:

  • 要运行的脚本就在当前目录，用英文的点.和反斜杠\加上脚本的名字，如下:
    .\show-powershellexample.ps1
  • 全路径执行，如下:
    C:\Users\SPPS\show-powershellexample.ps1
  • 不带后缀名的全路径执行，powershell会默认执行ps1后缀的脚本文件
    C:\Users\SPPS\show-powershellexample
  • 用Invoke-Expression cmdlet执行脚本:
    Invoke-Expression C:\Users\SPPS\show-powershellexample.ps1
  • 脚本路径包含空格时，用双引号包围住（没有空格也可以这么做，所以保险起见都用双引号）
    Invoke-Expression “C:\Users\SPPS\show-powershellexample.ps1”
  • 用&符号执行脚本
    & C:\Users\SPPS\show-powershellexample.ps1

配置文件

powershell有自己的预加载配置文件，比如某人最常用的工作路径是D:\temp，这样每次打开powershell console都要先执行cd D:\temp，虽然只是一条命令，但是很烦，我们就可以写在配置文件里。

  • C:\Windows\System32\WindowsPowerShell\v1.0\profile.ps1
    全局配置文件，对所有用户的所有powershell console生效，写在这里面的代码，只要powershell/PowerShellISE打开就会执行
  • C:\Users\SPPS\Documents\WindowsPowerShell\profile.ps1
    用户配置文件，对指定用户（这里就是SPPS）的所有powershell console生效，写在这里面的代码，只要这个用户打开powershell/PowerShellISE就会执行
  • C:\Users\SPPS\Documents\WindowsPowerShell\Microsoft.PowerShellISE_profile.ps1
    用户配置文件，只对指定用户（这里就是SPPS）的PowerShellISE生效，写在这里面的代码，只有这个用户打开PowerShellISE才会执行
  • C:\Users\SPPS\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1
    用户配置文件，只对指定用户（这里就是SPPS）的PowerShell生效，写在这里面的代码，只有这个用户打开PowerShell才会执行

这些配置文件默认是不存在的，需要我们手动创建并保存需要预处理的代码。
配置文件的任何更改只有重启powershell console才能生效,或者重新运行配置文件生效.
